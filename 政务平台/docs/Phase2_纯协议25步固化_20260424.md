## 五、铁律清单（刻在墙上）

### 铁律 1：signInfo 按域取值

```python
SIGN_INFO_NAME       = -252238669   # Phase 1（name/NameCheckInfo/NameSupplement/NameShareholder）
SIGN_INFO_ESTABLISH  = -1607173598  # Phase 2 establish（BasicInfo/MemberPost/MemberInfo/...）
```

### 铁律 2：字段名大小写字符级匹配

```python
cerno  # ✅ SlUploadMaterial 里是小写 n（Jackson 严格匹配）
cerNo  # ❌ 大写 N → Java DTO 反序列化成 null → NPE → A0002
```

### 铁律 3：Phase 2 establish flowData.busiId = None

Phase 1 的 busi_id 在 establish 域无效。step 12 `loadCurrentLocationInfo` 会返回新的 establish_busiId（保存在 `c.snapshot["establish_busiId"]`），后续 save 用它，**或直接传 None 让服务端推断**。

### 铁律 4：元数据必须从 load 响应剥离

load 响应的 busiData（或 list[0]）带 20+ meta 字段：
```python
{"flowData","linkData","processVo","jurisdiction","currentLocationVo",
 "producePdfVo","returnModifyVo","transferToOfflineVo","preSubmitVo",
 "submitVo","page","list","fieldList","busiComp","subBusiCompMap",
 "signInfo","operationResultVo","signRandomCode","extraDto","itemId"}
```
不剥 → `meta.flowData` 覆盖顶层 → 路径污染 → D0022。常量表在 `phase2_constants.BASICINFO_META_STRIP`。

### 铁律 5：加密字段按组件查真实样本

不同组件的加密字段不同。**永远不要猜**，直接对 mitm 的成功样本 diff 就知道哪个字段是密文、哪个是明文。

- BasicInfo：`entPhone` RSA（Base64 172 字符），`businessArea` 明文，`busiAreaData` URL-encoded JSON
- MemberBaseInfo：`cerNo` RSA，`homeAddress` 明文
- （其他组件待样本 fallback）

### 铁律 6：前端自动推进，协议化不需要手动 load 下一组件

SPA 检测到 `resultType=0` 会自动 load 下一组件。协议化只需**连续调下一 step 的 save**，不需要中间穿插 load。

## 九、边界声明（本阶段不 cover）

| 项 | 原因 |
|---|------|
| 电子签章（`ElectronicDoc`，step 25 之后） | 需要真实 CA 数字证书 U-key 硬件 |
| SSO 首次扫码 | 服务器强制二维码，不能完全后台 |
| 1151 有限公司 B 线（BenefitUsers iframe） | 当前个人独资 4540 不走此步，不影响 A 线；B 线需要单独深入逆向 |
| Case 广泛性 | 仅 `case_有为风.json`（4540 个人独资 / 玉林市容县）在真实数据下验证过；其他行业/地区/企业类型的 case 需要增量验证 |

## 十、风险与不确定性

| 风险 | 影响 | 缓解 |
|------|------|------|
| Step 15 body 42 keys 可能仍缺某关键字段 | 服务端 A0002 | 活 session 下逐字段 diff mitm 样本 |
| Step 18 MemberInfo 的 raw_member 需由 step 17 捕获，若 step 17 失败则 step 18 用骨架 | 可能报字段缺失 | step 17 失败时跳过 18 或人工补 raw |
| Step 21 文件上传：case.upload_materials 没配置时跳过 special 绑定 | 服务端可能仍要求材料齐全 | 调用方在 case 里配上传清单 |
| i18n 动态校验（如"认缴出资时间不得超过5年"） | Step 18 MemberInfo 可能 resultType=1 | `build_memberinfo_save_body` 默认 `investDate="2028-12-31"`（4 年内），可改 case.investDate |
| 同一浏览器 session 被多请求并发使用 | 幂等缓存已覆盖 5 分钟内同参数，但跨参数并发依然有风险 | 限流 `get_limiter()` 控制 |
