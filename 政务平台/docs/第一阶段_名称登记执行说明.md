# 第一阶段：名称登记 / 核名（单独完成）

> **目标**：在**不跑**「直至云提交」长主链的前提下，完成**名称自主申报、核名通过**，并按系统提示处理告知/材料下载（以实际页面为准）。  
> **与资料关系**：`docs/资料.txt` 中的企业名是**拟设主体**，用于在名称流程中填写；**不要求**跑脚本前「我的办件」里已出现该行。

---

## 1. 自动化能做什么（`--phase1-only`）

脚本 **`system/run_case_rongxian_to_yun_submit.py --phase1-only`**（或 `packet_chain_portal_from_start.py --phase1-only`）会：

- 从门户入口（`namenotice`）导航到**企业专区 / 名称登记子应用**等既定路径；  
- 安装 hook、打快照、写 JSON；  
- **不进入**「主循环直至云提交」——避免在核名未完成时空跑长链。

**不能代替**：在名称申报页内**逐字段填写、提交、处理弹窗、下载 PDF**——这些须在 **Chrome 里人工**完成（类人节奏、防风控）。

---

## 2. 推荐命令（第一阶段）

### A. 仅导航到名称登记子应用并快照（轻量）

```powershell
cd G:\UFO\政务平台
python system\run_case_rongxian_to_yun_submit.py --phase1-only
```

### B. 按案例尽量往下推：专区 → 须知 → guide → 自主申报 → 名称查重页填字号并尝试查重（推荐）

```powershell
python system\run_phase1_from_case.py
```

案例默认 `docs/case_广西容县李陈梦.json`。**仍须你在浏览器内**完成区划级联、验证码、以及具有法律效力的**最终提交**；脚本不代点。

前置：**已登录 9087**，Chrome Dev + CDP 可用（`scripts\launch_browser.py`）。

`--phase1-only` 结束后终端会打印 **「第一阶段专用模式」** 说明；退出码 **0** 表示「脚本按设计停住」，**不等于**核名已在业务上通过——以**页面与办件结果**为准。

---

## 3. 怎样算「第一阶段业务上完成」（人工验收）

以系统为准，常见包括：

- 名称查重/规则校验**通过**；  
- 按平台提示**下载或保存**告知类文件（若有）；  
- 在 **我的办件** 中**出现**与拟设名称**对应**的办件（时间有先后，**未立即出现**时继续跟页面提示，勿仅因列表暂无而判失败）。

记录可写入 `census_run_record` 或本机笔记。

## 4. 第二阶段（另跑，不在 `--phase1-only` 里做）

核名与办件就绪后，从**当前已打开的业务页**续跑主链至**云提交文案停点**（不点提交）：

```powershell
python system\run_case_rongxian_to_yun_submit.py --resume-current
```

（不带 `--phase1-only`。）
