# 扫码登录技术经验文档

> 最后更新：2026-04-22  
> 状态：**已验证成功**，QR 扫码 → token 全链路跑通

---

## 一、目标

用智桂通APP扫码登录，获取 9087 `Authorization` token，替代账号密码+滑块方式。

**核心脚本**：`system/login_qrcode.py`

---

## 二、最终架构

```
┌────────────────────────────────────────────────────────┐
│ Steps 1-5: 纯 HTTP（不开浏览器）                         │
│   tyrz 登录页 → QR 码 → 用户扫码 → submitLogin          │
│   结果：建立 SESSIONFORTYRZ (SSO session)                │
└────────────────────┬───────────────────────────────────┘
                     │ SESSIONFORTYRZ cookie
┌────────────────────▼───────────────────────────────────┐
│ Step 6: 浏览器 CDP                                      │
│   注入 cookie → enterprise-zone → SSO 链 → token        │
└────────────────────────────────────────────────────────┘
```

---

## 三、详细流程 & 接口

### Step 1：获取登录页

```
GET https://tyrz.zwfw.gxzf.gov.cn/am/auth/login?service=initService&goto={base64}
```

- **返回**：HTML 页面
- **获取**：`var csrfToken = 'xxx'`（单引号，用正则提取）
- **Cookie**：服务器设置 `SESSIONFORTYRZ`

### Step 2：获取 QR 码

```
POST https://tyrz.zwfw.gxzf.gov.cn/am/qrCode/getQrcode
Body: userType=1   (1=个人, 2=法人)
```

- **返回**：
  ```json
  {
    "data": {
      "qrCode": "data:image/png;base64,iVBOR...",
      "sessionId": "fe49bc5b855d41e9b4ec82daa1e693f3"
    }
  }
  ```
- QR 码是 base64 编码的 PNG 图片（约 4-6KB）

### Step 3：显示 QR 码

- 保存为 PNG 文件，`os.startfile()` 弹出图片查看器
- 用户用智桂通APP扫码

### Step 4：轮询扫码结果

```
POST https://tyrz.zwfw.gxzf.gov.cn/am/qrCode/checkQrCode
Body: random={sessionId}     ← 注意：参数名是 random，不是 sessionId
```

- 每 **5 秒**轮询一次，最多 12 次（60 秒）
- 返回值：
  | `res.data` | 含义 |
  |---|---|
  | `"1"` | 扫码成功 |
  | `"3"` | 用户不存在 |
  | 其他 | 登录失败 |

### Step 5：提交登录

```
POST https://tyrz.zwfw.gxzf.gov.cn/am/auth/submitLogin
Body: csrf_token={csrfToken}&qrCode={sessionId}
```

- `qrCode` 参数的值是 Step 2 返回的 `sessionId`
- **返回**：
  ```json
  {
    "result": true,
    "data": "https://tyrz.zwfw.gxzf.gov.cn/sso/oauth2/authorize?...",
    "userType": "1"
  }
  ```
- `data` 是 OAuth2 authorize URL（但纯 HTTP 访问会 404，见问题 1）
- 此步之后 `SESSIONFORTYRZ` cookie 已更新为已认证状态

### Step 6：浏览器 CDP 完成 redirect 链

1. 连接或启动浏览器（CDP 端口 9225）
2. 清除旧 auth 状态（cookies + localStorage）
3. 注入 `SESSIONFORTYRZ` cookie（domain=`.zwfw.gxzf.gov.cn`）
4. 启用 `Fetch.enable` 拦截 `ssc.mohrss.gov.cn`
5. 导航 `enterprise-zone` → SSO 自动完成 → Fetch 拦截 ssc → 302 到 6087 portal
6. 导航 `6087 portal` 确认 session（检查 `top-token`）
7. 导航 `entservice` → 9087 设置 `Authorization` 到 localStorage
8. 读取 `localStorage.getItem('Authorization')`

## 四、遇到的问题 & 解决方案

### 问题 1：tyrz `/sso/oauth2/authorize` 返回 nginx 404

**现象**：  
submitLogin 返回的 redirect URL 是 `https://tyrz.zwfw.gxzf.gov.cn/sso/oauth2/authorize?...`。  
无论用 Python requests 直连、带代理、还是浏览器 fetch()，都返回 **nginx 404**。

**排查**：
- 确认 DNS 解析正常（113.12.121.4）
- hosts 文件无特殊配置
- 加 Referer 无效、换 User-Agent 无效
- 代理 8080 返回同样 404（不同 Content-Length: 153 vs 555，但都是 nginx 404）

**根因**：  
tyrz 服务器的 nginx **没有配置 `/sso/` location**，只有 `/am/` 路径对外。  
`/sso/oauth2/authorize` 是内部服务，只有 server-side redirect 时由内部 nginx upstream 处理。

**解决**：  
放弃纯 HTTP 走 OAuth2 redirect 链。改用浏览器 CDP 导航 `enterprise-zone`，  
让 9087 SPA → 6087 → tyrz 的 **server-side redirect 链**自然完成。  
浏览器页面导航不受 `/sso/` 路径限制（因为是 302 redirect 由 server 内部处理）。

---

### 问题 2：HTTP 请求 tyrz 超时

**现象**：  
Python requests 请求 `tyrz.zwfw.gxzf.gov.cn` 一直卡住不返回。

**根因**：  
系统配置了代理（可能是 Fiddler/mitmproxy），requests 默认走代理。

**解决**：
```python
s.proxies = {"https": None, "http": None}  # 显式绕过代理
s.timeout = 10
```

---

### 问题 3：6087 authLogin 返回 JSON 而非 302

**现象**：  
`GET 6087/TopIP/sso/authLogin?authType=zwfw_guangxi` 返回：
```json
{"msg":"没有权限","code":"-8","data":{"redirect_uri":"..."}}
```

**根因**：  
Session 的默认 `Accept` 头是 `application/json`。6087 服务器根据 Accept 判断：
- JSON Accept → 返回 JSON 错误
- HTML Accept → 返回 302 redirect

**解决**：
```python
browser_headers = {"Accept": "text/html,application/xhtml+xml,application/xml;q=0.9,*/*;q=0.8"}
r = s.get(url, headers=browser_headers)
```

---

### 问题 4：Fetch 事件堆积导致浏览器挂起

**现象**：  
导航后用 `time.sleep()` 等待，期间 Fetch.requestPaused 事件在 WebSocket 缓冲区堆积，  
浏览器因为 Fetch 请求未被 continue/fulfill 而挂起。

**根因**：  
sleep 期间没有人读取 WebSocket 消息，Fetch 事件无人处理。

**解决**：  
用主动消息循环替代 sleep：
```python
ws.settimeout(3)  # 3秒超时
while time.time() < deadline:
    try:
        msg = json.loads(ws.recv())
    except:
        continue  # 超时重试
    if msg.get("method") == "Fetch.requestPaused":
        # 处理 Fetch 事件
        ...
```

---

### 问题 5：重复登录触发风控

**现象**：  
每次调试路由都重新扫码登录，短时间内多次登录可能触发 tyrz 风控。

**教训**：  
- **登录只做一次**，后续用已建立的 session 去探测路由
- 保持类人节奏（导航间加 2-5 秒延时）
- 不穷举路径，先分析再行动

---

### 问题 6：checkQrCode 参数名不是 sessionId

**现象**：  
参数传 `sessionId=xxx` 无效。

**根因**：  
JS 源码中参数名是 `random`：
```javascript
ajaxFun("../qrCode/checkQrCode", null, true, null, { random: text }, ...)
```

**解决**：
```python
s.post(QR_CHECK, data={"random": session_id})
```
