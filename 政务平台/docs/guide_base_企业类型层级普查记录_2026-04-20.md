# guide/base 企业类型层级普查记录（2026-04-20）

## 1. 目的

为后续 **LLM 单步协同政务自动化框架** 提供可复用的页面知识，将 `guide/base` 页面中“企业类型”选择器的层级结构先做一次实网普查并固化记录，避免后续在相同步骤反复盲试。

本记录面向：

- `guide/base` 页面自动化
- 企业类型选择器识别与回放
- 后续更多页面的“先普查、再抽象、后推进”框架化沉淀

---

## 2. 普查页面

- **页面**：`name-register.html#/guide/base`
- **业务位置**：步骤 1「请选择您要办理的市场主体类型」
- **关键控件**：
  - 上方高频单选：`内资有限公司 / 农民专业合作社 / 个人独资企业 / 个体工商户 / 分支机构 / 全部企业类型`
  - 下方真正层级选择器：`placeholder=请选择企业类型` 的 `tne-data-picker`

---

## 3. 关键发现

- **[真正的层级控件]** 不是上方 `全部企业类型` 单选本身，而是下方 `请选择企业类型` 的 `tne-data-picker`。
- **[上方单选的作用]** 是把企业类型模式切到“全部企业类型”。
- **[真正的编号层级数据]** 来自企业类型 picker 的内部树数据，而不是地址区划 picker。
- **[本轮有效结果]** 已稳定采到 `7` 个根节点及其下级编号/名称。
- **[层级深度]** 当前实网观察到的是 **1-3 级有效层级**；本轮 **未观察到第 4 级子项**。
- **[当前主线结论]** 按现有 `docs/case_广西容县李陈梦.json`，目标公司当前应走 `1000 -> 1100` 路线。

---

## 7. 对框架沉淀的价值

这份记录后续可直接用于：

- **[状态识别]** 判断当前是否已切到“全部企业类型”模式
- **[动作规划]** 先点高频单选，还是打开层级 picker
- **[编号回放]** 按编号而不是只按中文名做稳态选择
- **[证据落盘]** 每个页面先普查一次，后续动作都复用知识库
- **[错误归因]** 区分“点错控件”与“picker 数据未打开”两类问题
