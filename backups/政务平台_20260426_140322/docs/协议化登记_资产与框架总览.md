## 一、我们在做什么

把广西经营主体登记平台（`zhjg.scjdglj.gxzf.gov.cn:9087` / `icpsp-api`）的**企业设立登记**全流程，从"浏览器里人工点击 + 填表"转成"纯 HTTP 协议调用"。

**总体范围**：
- **Phase 1 名称登记**：7 个 API 拿到 `busiId`
- **Phase 2 设立登记**：11 个组件（BasicInfo → MemberPost → MemberPool → MemberInfo → ComplementInfo → TaxInvoice → SlUploadMaterial → BusinessLicenceWay → YbbSelect → PreElectronicDoc → PreSubmitSuccess）

**已达成里程碑**：
- ✅ Phase 1 全链路协议化（`phase1_protocol_driver.py`）
- ✅ Phase 2 A 线（4540 个人独资）**推进到 PreSubmitSuccess 云提交停点**（100%）— 见 `@/UFO/政务平台/docs/Phase2_云提交停点达标_20260424.md`
- ✅ Phase 2 B 线（1151 有限公司）推进到 ComplementInfo（~30%，卡在 BenefitUsers iframe）
- ✅ **27 个 FastAPI 端点**（Phase 1/2 + 认证 + QR 扫码 + 办件列表 + 进度查询 + 字典实时 + sysParam + 调试）— 见 `@/UFO/政务平台/docs/API调用手册_Phase1_Phase2.md`
- ✅ Session 自愈 / Token 静默续期 / QR 分步登录 / 幂等缓存 / D0029 限流降级
- ✅ **API 覆盖率 ~98%**（核心用户流 100%；仅剩"办件作废"因危险延后）— 见 `@/UFO/政务平台/docs/API未覆盖资产盘点_20260424.md`

**当前卡点**：
- **新案例**的 BasicInfo 纯 HTTP save 复现（session/时序/currCompUrl 依赖）— 昨天通过 CDP+浏览器已突破，但纯 HTTP 单次复现仍未稳定
- B 线的 BenefitUsers iframe 加载问题

## 六、方法论（按主题）

### 6.1 D0022 / A0002 / D0021 调试

- `@/UFO/政务平台/docs/突破D0022越权_协议层字段合同方法论_20260422.md` — 三层洋葱（浏览器指纹头 / body 结构 / signInfo 魔数）
- `@/UFO/政务平台/docs/Phase2_阶段总结_20260423.md` 第二节 — 8 个典型问题解法

### 6.2 加密字段处理

- `entPhone` / `cerNo` / `email` / `mobile` 等在 encryptFields 列表里的字段
- SPA 通过 `$util.deepClone(busiData, true, encryptFields)` 自动 AES 加密
- 纯协议需手动预加密，或保持明文（部分接口容忍，需实测）

### 6.3 前端自动推进

- save 成功（`resultType=0`） → SPA 自动 load 下一组件 → URL 变化
- 协议化时**只需保证每步 save 成功**，前端自推进

### 6.4 元数据污染防御

- load 响应带 20+ meta 字段（flowData/linkData/processVo/fieldList/…）
- save 时必须剥离，否则覆盖顶层 flowData → 路径污染 → D0022
- 过滤表见 `@/UFO/政务平台/system/phase2_constants.py` 的 `BASICINFO_META_STRIP`

### 6.5 Session 恢复

- `session_expired` (GS52010103E0302) → 扫码重登 → `sync_icpsp_session` 同步 cookie 到 pickle
- Edge 启动不带 `--proxy-server`（mitm 停后代理残留会导致所有请求 ERR_EMPTY_RESPONSE）

## 七、核心魔数和铁律（便于速查）

### 铁律 1：signInfo 魔数

```python
SIGN_INFO_NAME       = -252238669   # Phase 1 域（NameCheckInfo/NameSupplement/NameShareholder）
SIGN_INFO_ESTABLISH  = -1607173598  # Phase 2 establish 域（BasicInfo/MemberPost/MemberInfo/...）
```

### 铁律 2：字段名大小写字符级匹配（A0002 常见原因）

```python
cerno  # ✅ SlUploadMaterial 小写 n
cerNo  # ❌ 会被 Jackson 丢成 null → NPE → A0002
```

### 铁律 3：Phase 2 establish busiId 是 null

establish/loadCurrentLocationInfo 传的 body 里 `busiId=null`，只带 `busiType=02_4 + entType + nameId`。Phase 1 的 busiId 在 establish 里无效。

### 铁律 4：matters/operate 激活

从名称提交到设立登记的过渡：必须先调 `matters/operate btnCode=108 dealFlag=before/operate`，服务端才会把会话状态切到设立域。

### 铁律 5：前端验证规则可能是服务端动态生成

"认缴出资时间不得超过5年" 这种错误消息是 i18n key（`register.msg.RealInvestDateBeforeFiveYear`），规则由服务端返回，静态 JS chunk 里搜不到。
