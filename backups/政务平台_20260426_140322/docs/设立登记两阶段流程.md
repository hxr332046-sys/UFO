# 设立登记：两阶段流程与自动化停点

> **目的**：把业务上「先核名、再办件」拆清楚，避免脚本把两阶段混成一条无状态乱点链。  
> **自动化边界**：第二阶段推进到 **出现云提交相关界面/文案** 即视为任务完成，**不执行云提交、不进入云处理**（与 `docs/实网开发操作纪律_单步与风控.md` 一致）。

---

## 第一阶段 — 名称登记（核名）

| 事项 | 说明 |
|------|------|
| **目标** | 确定拟用名称是否可用（名称查重 / 规则校验）。 |
| **成功** | 流程通过；系统通常会**附送两个文件**（以实际页面为准，用于下载或后续引用）。 |
| **失败** | 名称不可用会有**明确提示**（需读 UI / 接口反馈，再改名称或策略）。 |

此阶段在门户/名称子应用中完成，**不等价于**已进入完整设立表单主流程。

**单独跑第一阶段（自动化辅助 + 人工核名）**：见 **[第一阶段_名称登记执行说明.md](./第一阶段_名称登记执行说明.md)**（`--phase1-only`）。

---

## 第二阶段 — 从「我的办件」接续办理（资料录入与上传）

第一阶段通过后，**不能**假设同一标签页已直接进入长表单；通常需要：

1. 进入 **我的办件中心** → **我的办件**；  
2. 在列表中找到 **与第一阶段已核准名称相对应** 的那条办件；  
3. 点击 **继续办理**，**正式进入**第二阶段的填报向导。

第二阶段在 UI 上表现为**多步向导**（顺序以页面为准，常见包括）：

1. 基本信息  
2. 成员架构  
3. 成员信息  
4. 补充信息  
5. 税务信息填报  
6. 材料补充（上传类步骤集中区）  
7. 营业执照领取  
8. 云帮办流程模式选择  

**最后一步**会到达 **云提交** 相关环节（云帮办/云处理入口）。  

**本仓库任务完成标准**：自动化或人机协同推进到 **云提交界面已就绪（文案/按钮可见）** 即 **停止**，**不点击**最终提交、不把业务送入云处理队列。

**验收与复盘**：何谓「第一阶段完成 / 第二阶段完成」、如何对照「我的办件」列表列，见 **[阶段验收清单.md](./阶段验收清单.md)**；运行结果以 JSON 中 **`phase_verdict`** 为准。
