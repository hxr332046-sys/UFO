## 二、未 API 化的"确定技术" — 按优先级

### ⭐⭐⭐ 优先级 1：高价值 + 技术已验证

#### 2.1 扫码登录 + Token 刷新（`system/login_qrcode_pure_http.py`）

已验证能力：
- `check_token_alive()` — 32 字符 Authorization 是否能调通 getUserInfo（部分已通过 `/api/auth/status?probe=true` 暴露）
- **`refresh_token()`** — 静默续期（用已存 SESSIONFORTYRZ 走 SSO Steps 2-4），**~2 秒完成不需要扫码** ← **未 API 化**
- **`full_login()`** — 全新扫码登录（QR + 4 步 SSO，30 秒需人工）← **未 API 化**
- `ensure_token()` — 智能获取（有效直接返，否则先静默，最后扫码）

建议新 API：
```
POST /api/auth/token/refresh      → 静默续期（优先）
POST /api/auth/qr/start           → 返回 {sessionId, qr_image_base64, expire_sec}
GET  /api/auth/qr/status?sid=X    → 轮询 {scanned, confirmed, token}
POST /api/auth/token/ensure       → 组合：先 refresh，失败再给 QR
```

**为什么重要**：当前每天第一次用必须手动扫码 + 手动 `/session/recover`。API 化后前端可直接显示二维码。

---

#### 2.2 办件进度查询（`establish/loadCurrentLocationInfo`）

已验证：Phase 2 step 12 的 `establish/loadCurrentLocationInfo` 能告诉你 `busiId` 当前在哪个组件（`flowData.currCompUrl` + `status`）。

建议新 API：
```
GET  /api/phase2/progress?busi_id=X           → 当前 currCompUrl + status + 已完成组件数
GET  /api/phase2/progress?busi_id=X&detail=1  → 带 busiData 快照
```

**为什么重要**：断点续跑必备。现在只能用 `POST /api/phase2/register start_from=12,stop_after=12` 绕过去查。

---

#### 2.3 "我的办件"列表（`matters/search`）

已验证：`GET /icpsp-api/v4/pc/manager/mattermanager/matters/search` 返回当前用户所有在办业务（昨天记录里 `matters/search` 已经调通过）。

建议新 API：
```
GET /api/matters/list                         → 我的办件列表（含 busiId/nameId/状态/创建时间）
GET /api/matters/detail?busi_id=X             → 单个办件详情
```

**为什么重要**：你让我帮你跑时总要人工填 busiId/nameId，有这个 API 后调用方能自动发现当前可推进的办件。

---

### ⭐⭐ 优先级 2：有 API 但不完整

#### 2.4 经营范围实时搜索（`getThirdleveLBusitermList`）

现状：`/api/business-scope?entType&busiType` 只从**本地 census** 返结果。

已验证：`GET /icpsp-api/v4/pc/common/busiterm/getThirdleveLBusitermList?keyWord=软件&indusTypeCode=6513` 返回平台最新字典（`I3006` 软件开发等）。Phase 1 step 7 就靠它。

建议新 API：
```
GET /api/business-scope/search?keyword=软件&industry_code=6513
GET /api/business-scope/search?keyword=软件&live=1  → 强制走平台实时 API（而非本地 census）
```

**为什么重要**：平台会更新行业编码（`stateCo=1` 可能变 `3`），本地 census 过期会触发 rt=1 警告。

---

#### 2.5 系统参数（sysParam）快照刷新

现状：`dashboard/data/records/sysparam_snapshot.json` 是 2026-04-23 的快照（含 `aesKey=topneticpsp12345` + `numberEncryptPublicKey`）。归档脚本 `_phase2_extract_sysparam_keys.py`（已归档）能重新拉取 957 条。

建议新 API：
```
POST /api/system/sysparam/refresh   → 实时拉 957 条系统参数更新快照
GET  /api/system/sysparam/snapshot   → 读当前快照（含 aesKey/RSA 公钥等关键密钥）
```

**为什么重要**：如果平台换 aesKey，本地加密全部失效。需要能一键刷新。

---

### ⭐ 优先级 3：调试/开发辅助

#### 2.6 mitm 样本查询

现状：`dashboard/data/records/mitm_ufo_flows.jsonl` 有 1791 条流量，`establish_save_samples/` 有 3 个提取好的样本。但**查询要手写 Python**。

建议新 API：
```
GET /api/debug/mitm/samples?api=BasicInfo&opeType=save&limit=5
GET /api/debug/mitm/latest?api_pattern=BasicInfo/op
```

**为什么重要**：开发/调试时快速对比 body 结构。对外服务不需要。

---

#### 2.7 办件删除 / 作废（`matters/operate btnCode=103/109`）

已验证：`btnCode=103` 删除办件，`btnCode=109` 作废名称。文档里明确标 **"危险"**。

**已实现** `POST /api/matters/delete`：
```
# 单个删除（confirm = busi_id）
POST /api/matters/delete?busi_id=X&confirm=X

# 批量删除（按企业名搜索，confirm = 'DELETE'）
POST /api/matters/delete?search=昱铭&confirm=DELETE
```

底层协议：`matters/operate` + `btnCode=103`
- Step 1: `dealFlag=before` → resultType=2 "此操作将永久删除, 是否继续?"
- Step 2: `dealFlag=operate` → resultType=0 "删除成功！"
- **注意**：`dealFlag=after` 不会删除，只是重复返回确认提示！

## 六、剩余差距（可选第三批）

### 6.1 ⭐⭐ 经营范围实时搜索（~1 小时）

```
GET /api/business-scope/search?keyword=&industry_code=&live=0
  - live=1 → 走平台 /getThirdleveLBusitermList 实时查
  - live=0 → 走本地 census 产物（默认）
```

**为什么有价值**：本地 census 是 2026-04 抓的快照，若平台增删行业条目或改 `stateCo`，离线查询会失配触发 Phase 1 step 7 的 `rt=1 警告`。

### 6.2 ⭐ 系统参数刷新（~30 分钟）

```
POST /api/system/sysparam/refresh   → 重新拉 957 条 sysParam 更新本地 snapshot
GET  /api/system/sysparam/snapshot   → 读当前快照（含 aesKey / RSA 公钥等）
```

**为什么有价值**：加密密钥一旦更新，本地所有 AES/RSA 计算都会出错。

### 6.3 ⭐ mitm 样本查询（~30 分钟）

```
GET /api/debug/mitm/samples?api=BasicInfo&opeType=save&limit=5
GET /api/debug/mitm/latest?api_pattern=BasicInfo/op
```

**为什么有价值**：调试/对比 body 结构时无需手写脚本。

### 6.4 ⚠️ 办件作废（危险，可延后）

```
POST /api/phase1/cancel?busi_id=X&confirm=<busi_id>
POST /api/phase1/delete?busi_id=X&confirm=<busi_id>
```

**为什么要做**：调试时累积的 busiId 会污染"我的办件"列表。**但是**：需要二次确认机制 + 仅限开发环境暴露。

### 6.5 不打算做的

| 技术 | 理由 |
|------|------|
| 加密/解密 API | 仅内部使用 |
| mitm 抓包本体 | 开发工具，本机手启 |
| CDP 浏览器启动 | 本地操作，API 无法代劳 |
| 电子签章 / 真正云提交 | CA U-key 硬件 |
| 1151 BenefitUsers iframe | 技术**未攻克** |
