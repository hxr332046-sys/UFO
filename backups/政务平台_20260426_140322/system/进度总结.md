## 三、当前阻塞问题

### ❌ 4个验证错误无法通过

保存时触发验证，以下4个字段报错：

| 错误信息 | 组件类型 | 问题根因 |
|----------|----------|----------|
| **请选择企业住所** | tne-data-cascader | 级联选择器需要交互式选择，`$emit('input')`不触发验证通过 |
| **请选择生产经营地址** | tne-data-cascader | 同上 |
| **请选择行业类型** | tne-select (tree模式) | 树形下拉选择器，`$emit('input','I65')`不触发内部状态更新 |
| **请填写经营范围** | 按钮触发iframe对话框 | 对话框在iframe中，`display:none`，内容为空 |

### 问题分析

1. **cascader**: `tne-data-cascader`是自定义组件，不是标准`el-cascader`。`$emit('input')`设置了值但组件内部状态未同步，验证仍失败。需要模拟用户点击级联面板选择省/市/区。

2. **行业类型**: `tne-select`是树形下拉，选项为懒加载。已确认选项列表：
   - `[A]农、林、牧、渔业` ~ `[T]国际组织`
   - `[I]信息传输、软件和信息技术服务业` → 子选项 `[63]电信`、`[64]互联网`、`[65]软件和信息技术服务业`
   - 点击展开`[I]`后选择`[65]`，但点击后Vue组件值未更新

3. **经营范围**: 点击"添加规范经营用语"按钮后，iframe中的对话框`display:none`，body内容为空。对话框是`tni-dialog`自定义组件，可能需要通过Vue方法打开而非DOM点击。

## 四、技术架构发现

### SPA结构
- **框架**: Vue 2 + Element UI + 自定义组件(tne-*)
- **路由**: Vue Router，动态路由通过组件方法注册
- **状态**: Vuex store (`login/SET_TOKEN`, `login/SET_USER_INFO`)
- **认证**: `top-token` + `Authorization` header

### 关键API
| API | 用途 |
|-----|------|
| `notHmEnterpriseType` | 获取企业类型列表(establish页面) |
| `selectBusinessModules` | 获取业务模块 |
| `searchConsultPhonePageData` | 咨询电话 |
| `operationBusinessData` | 保存表单数据(POST) |
| `loadbusinessData` | 加载表单数据 |

### 关键Vue组件方法
| 组件 | 方法 | 用途 |
|------|------|------|
| `without-name` | `toNotName()` | 跳过名称选择，进入establish |
| `establish` | `nextBtn()` | 选择企业类型后跳转namenotice |
| `establish` | `checkchange(e,t)` | 设置radioGroup |
| businessDataInfo组件 | `save(btn,cb,sign)` | 保存表单 |
| businessDataInfo组件 | `handleSteps(step)` | 步骤导航 |
| businessDataInfo组件 | `handleStepsNext(step)` | 跳到指定步骤 |
| businessDataInfo组件 | `jumpWidget()` | 跳转到当前widget |
| businessDataInfo组件 | `next()` | 下一步 |

### 步骤列表(7步)
1. 信息填报
2. 信息确认
3. 信息预审
4. 电子签章
5. 信息审核
6. 告知书
7. 营业执照
